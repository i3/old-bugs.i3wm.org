Description

If it's possible, I think it would be very useful to have an option to hide all floating windows at once.
It is a feature that I used, while using other WMs.
For example, if you have multiple floating windows and you have to have them displayed on different sides of the workspace (side-by-side), and hide them in one instant, then again, show them. .
I find scratchpad very useful, but as you know, if you have multiple floating windows which you need to display in one instant (and those windows aren't in one container), it's too much work.
If you use scratchpad, it will center everything, and you then need to move it on the side just to be able to see both (floating) windows at the same time, also, you'll need to switch focus to the tiling window and then scratchpad show, to display second (floating) window (because previous one will disappear if you don't do it). Some other example, (which I often encounter is,) if MPlayer is in floating window, using scratchpad will resize it, default video size would be lost. .
There're plenty other use-cases. I'm certain that other people will find it useful.
Thank you for such a useful software. ;)

comment:3 Changed 2 years ago by FabiÃ¡n Ezequiel Gallina <fabian@â€¦>

Attached is a working implementation for this.

The concepts behind it are:

1) Never mess with scratchpad windows (their visibility is never affected by the toggle).
2) Make scratchpad behavior consistent when toggling floating windows visibility (scratchpad windows are always over floating windows, even when floating windows are re-displayed).
3) Floating visibility toggle is local to a single workspace.
4) Focus must be relocated when the floating windows are hid and the focus was in one of them.
