comment:3 in reply to: ↑ 2 Changed 5 months ago by fabianorosas@…

While collecting the information you requested, I realized I wasn't using the command you provided in the man page. Everything is ok if I suspend with i3lock && echo mem > /sys/power/state. The problem seems to be the way xfce deals with screen locking. Sorry to have bothered you.

PS: Just in case this pops up again, the problem arises when doing a workaround to use a unsupported screenlocker with xflock4:

Since xflock4 tries to run one of xscreensaver, gnome-screensaver, slock or xlockmore, what some people suggested online is to echo i3lock > /usr/bin/slock. This approach will raise the problem reported ​here when using the -i flag (presumably because it takes some time to fetch the file), and the user presses the Menu key.
