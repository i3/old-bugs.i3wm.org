Description

Please add a toggle to ignore _NET_WM_STATE_FULLSCREEN on a window-by-window basis, similar to float [enable | disable | toggle]. If we could configure i3 to ignore the _NET_WM_STATE_FULLSCREEN for Firefox, Chromium, etc, then a user could hit F11 from within such an app, hiding unnecessary widgets while keeping the window tiled. Then if the user really want the app full screen, they could trigger i3's fullscreen mode for the window.

A command like the following would be great:

ignore_fullscreen_hint [enable | disable | toggle]

for_window [class="^Google-chrome"] ignore_fullscreen_hint enable

Thanks. I played with this some more and can get the window in the desired state, but it's confusing. The state I am aiming for is: tiled as a window, but "fullscreen" from the browser's perspective.

To make this work, I do the following:

  • Press F11 in the browser. I3 immediately makes the browser window fullscreen. The browser window hides its widgets.
  • Press $mod+f. I3 restores the window to tiled state. The browser keeps the widgets hidden.
  • The i3 fullscreen toggle is jarring and requires an extra $mod+f each time to restore the tile.

Ideally I was hoping for:

  • Press F11 in the browser. I3 does nothing. The browser hides its widgets.
