Make i3-dmenu-desktop startup faster

Reported by: gereon.kremer@… Owned by: michael
Priority: major Component: general
Version: 4.4 Keywords:
Cc: Is it reproducible?: Always
git version: 4.4 Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

i3-dmenu-desktop takes some 100ms to startup. This is not really slow, but in this case already annoying: When you start typing down your application directly after launching i3-dmenu-desktop, you can very well be faster than the launcher and the first few keystrokes are lost.
This is on a system with <200 .desktop files in /usr/share/applications/ on a SSD... so I guess it might be even worse on other systems.

At the same time, I see a relatively easy approach to get a major speedup: As far as I understand the script (I never used perl...), it parses all desktop-files everytime.
Couldn't we just do it once and cache the result somewhere in /tmp/?

comment:2 Changed 2 years ago by gereon.kremer@…

I just did a bit of profiling with Time::HiRes.

I measured from the beginning of the script (before the includes and after them, but this turned out to make no difference at all. Hence I guess the perl startup is pretty fast...) to line ~260 where dmenu is about to be started.
With 169 desktop files, it took about 44ms, with none left, it took about 4ms.
What remains is the time perl takes to load the script itself and the dmenu startup...

If we'd cache the result, we could save the call to perl altogether and directly call dmenu... I'd consider this consequent, considering how you designed i3status avoiding every syscall whereever possible ;-)
