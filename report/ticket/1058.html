i3-input malfunctions if primary monitor is not at 0x0

Reported by: ryzion@… Owned by: michael
Priority: major Component: general
Version: 4.5.1 Keywords:
Cc: Is it reproducible?: Always
git version: 4.5.1 (2013-03-18, branch "tags/4.5.1") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

Using other positions than 0x0 for the primary monitor induces i3-input to malfunction. i3-input will still accept inputs but the input bar is not visible.

Tested command: bindsym $mod+n exec i3-input -F 'rename workspace to "%s"' -P 'New name: '

comment:2 Changed 20 months ago by jebaum@…

this behavior is expected, though obviously not ideal. the location of the i3-input bar is hard coded at 50x50, so if you don't have a monitor there you can't see it, but can still type into it. I'm currently working on a patch for this, and in the meantime you can edit the source as a workaround. Around line 405 of i3-input/main.c there is this line:

50, 50, 500, font.height + 8, /* dimensions */

change the first two numbers to whatever coordinates you want the input bar to appear at.
