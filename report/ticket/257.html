Keeping log alive (compatibility with logrotate)

Reported by: marfig@… Owned by: michael
Priority: major Component: feature-request
Version: Keywords: log
Cc: Is it reproducible?: Always
git version: Operating system: Linux
X11 program: Distribution / flavor:
Your IRC nick:

Description

When logrotate performs a rotation, the original log file is deleted and a new one created, as per settings in my logrotate.conf (see below). This action however means that I have to restart i3 to start feeding the new log file.

It would be great if i3 either:
a) detects the opened file buffer can no longer be reached and reopens it next time there is the need to output to it.
b)Or the user can manually (i3-msg, IPC) point i3 to the log file and thus force it to reopen the file.

#logrotate.conf
compress

$HOME/.i3/i3log {
	rotate 2
	daily
	create
}

Change History (5)

comment:1 Changed 5 years ago by michael

While I would like that to work, re-opening the fd sounds much easier than it is. As you might have noticed, i3 logs to stdout/stderr. You don’t pass it a filename, you redirect the file descriptors using your shell (in xsession/xinitrc). i3 does not even know the filenames. So, basically, one would need to build logging to specific files right into i3.

comment:4 Changed 4 years ago by mark.kusch@…

From man logrotate.conf:

      copytruncate
              Truncate the original log file in place after creating a copy, instead of moving the old log file and optionally  cre‐
              ating a new one.  It can be used when some program cannot be told to close its logfile and thus might continue writing
              (appending) to the previous log file forever.  Note that there is a very small time slice between copying the file and
              truncating  it,  so some logging data might be lost.  When this option is used, the create option will have no effect,
              as the old log file stays in place.

I would not think about handling such things when other programs already can.

comment:5 Changed 4 years ago by michael

  • Resolution set to wontfix

Indeed, thanks for your hint. I just tested the following logrotate config file and it works great:

/home/michael/i3/i3log {
	daily
	dateext
	rotate 7
	compress
	copytruncate
}
