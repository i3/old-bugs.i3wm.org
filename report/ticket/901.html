stacked window title bars are counter-intuitive

Reported by: tanguy+i3wm.org@… Owned by: michael
Priority: major Component: general
Version: 4.4 Keywords:
Cc: Is it reproducible?: Always
git version: 4.4 (2012-12-12, branch "tags/4.4") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

In stacking mode, window title bars are all displayed on top of the container, with the active window title highlighted. However, that active window title bar can be separated from the window itself by the title bars of other inactive windows below it.

This is kind of counter-intuitive, because one can usually expect to see window title bars to be attached to the corresponding windows.

It would be more intuitive if the title bars of windows that are below the active ones were put below that window, the way it is done in wmii for instance.
