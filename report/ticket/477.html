On first startup i3 misplaces windows.

Reported by: nutron Owned by: michael
Priority: minor Component: general
Version: 4.0 Keywords: workspace focus
Cc: Is it reproducible?: Always
git version: 4.0.1-223-g2f116df Operating system: Linux
X11 program: Distribution / flavor: Debian Sid
Your IRC nick: nutron

Description

Fresh install of i3 v.4.0.1 on debian sid. The config used is the default that ships with the package. I run three screens with the Nvidia binary blob driver.

Upon login from slim, i3 starts and places the mouse pointer on screen 1. Pressing Alt-D brings up dmenu on the correct screen (workspace 1), but after typing the application name to start up, in this case chromium, it places the open application on workspace 3.

The interesting caveat is that if unclutter is installed and called via .xsession, this behaviour disappears and works as intended.

The other noticable side effect that may be related is that a simple Alt-1 through Alt-9 keypress changes the workspaces, but the mouse pointer does not follow the focus unless the workspace existed prior. This behaviour shows up without unclutter installed. I will test with it installed to see if the behaviour still exists.
