comment:2 Changed 2 years ago by moritz.heiber@…

Essentially, this is about convenience: Almost all NetWM, session aware window managers implement a command line modifier called '--replace' [1], which basically tells the running session manager to quit whatever window manager is running at the current time and replace it with an instance of itself in the user's current session. This stays on throughout session cycles, thus making it much easier for one to run i3 instead of another window manager (in my instance: xfwm4) without even having to touch a single configuration file.

Right now, one needs to edit configuration files and/or add autostart entries in order to get i3 integrated into an otherwise session driven environment.

Pertaining to your comment about X session management: I was under the impression that session managers are an everyday convenience for most users of desktop environments these days. As far as I know almost any distribution is using one (gnome-session, kwin, xfce4-session) to manage their user's X sessions. Did I refer to the wrong reference documentation?

[1] ​http://git.xfce.org/xfce/xfwm4/tree/src/main.c => replace_wm
