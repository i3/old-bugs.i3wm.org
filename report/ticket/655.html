Description

Hi,

I am moving my comments from bug 550 to new one because I think it is more correct to explain this problem separately.

Running on i3 git commit 78cc929cdaaf75abea297241fd88977bd58b6e19
(also tried the same on all stable versions of i3wm).

Background: I am working on laptop, which I bring with me across several offices and home locations.

Every place where I work has a different subset of displays: sometimes I work on laptop only (in car for example), sometimes I connect external main (primary) display only, and sometimes I use both laptop (LVDS1) and external (HDMI1) displays.

The problem which I have is the following: every time when primary changes its location, almost all tray applictations die.

How to reproduce. My external monitor connects as:

xrandr --output HDMI1 --auto --primary --output LVDS1 --off

My external monitor disconnects as:

xrandr --output HDMI1 --off --output VGA1 --off --output LVDS1 --auto

Both actions result in tray applications to die similar way.

BUT, if I just add or remove non-primary display, all is well and there are no bug.

My tray applications list:

skype pidgin wicd-client dropbox parcellite

Exactly, the following happens:

1) Skype stays on
2) pidgin stays on only on screen, tray icon disappears. If pidgin is iconized,
I can not get it on screen again because there are no control for it, so have to kill
& restart.
3) wicd-client disappears, and even if it was on screen during
screen change it would exit
4) dropbox exits as well.
