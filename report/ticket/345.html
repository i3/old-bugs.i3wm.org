movement across unsplit containers should split and merge target

Description

Because there are no visual indicators for containers, clients moving "over" a unsplit container should create a temporary container to move into with some default behaviour. It's very confusing to see a client move across and merge into some columns/rows but then skip over others. This forces the user to move to the column/row and split it, then go back to the original client and merge if they want that behaviour.

Additional keystrokes and time would be saved with this sort of behaviour. Should the Window Manager expect the user keep track of each invisible container on the screen?
