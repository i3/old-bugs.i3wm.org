Sorry for taking so long til addressing this bugreport.

Yes, the generated configfile is a bit wrong when running under Xephyr, this is because the keyboard mapping for the cursor keys (left/right/up/down) are broken for some reason.

This is unrelated to not being able to spawn a terminal, though. I assume that you (or the slackbuild?) are missing a dependency on any terminal emulator which is supported by i3-sensible-terminal (by default: urxvt, rxvt, terminator, eterm, aterm, xterm, gnome-terminal, roxterm, although distributions might modify this list). Could you check what happens when you run DISPLAY=:1 i3-sensible-terminal from the system on which you have Xephyr running on (given that :1 is the display of Xephyr)?
