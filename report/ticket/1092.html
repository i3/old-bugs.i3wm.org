#1092 new defect

Wrong documentation about "short_text" JSON key in i3bar.

Reported by: hannes.koerber@… Owned by: michael
Priority: major Component: general
Version: 4.6 Keywords:
Cc: Is it reproducible?: Always
git version: 4.6 (2013-08-07, branch "tags/4.6") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

docs/i3bar-protocol mentions a "short_text" key that would be used instead of the "full_text" key "in case the status line needs to be shortened because it uses more space than your screen provides".

When looking through the source code of i3bar I could not find any traces of such a key, the git logs did not show anything either. When I tried using the key, it was ignored and "full_text" was used regardless of remaining space.

Was this a feature in past versions or did I miss something? If not, I would update the documentation accordingly.
