#751 closed enhancement (fixed)

"exit" operation should ask for confirmation

Reported by: felicitus@… Owned by: michael
Priority: minor Component: general
Version: 4.2 Keywords:
Cc: Is it reproducible?: Always
git version: Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick: Felicitus

Description

The default keybinding for exit is:

bindsym $mod+Shift+E exit

As discussed in IRC, this should ask for confirmation. It is quite possible to hit that combination by accident (e.g. GIMP uses Ctrl+Shift+E, and in case you mis-hit your keyboard, this can kill your whole session).

A slightly better approach would be to introduce a new action, something like "exit-without-confirm" for those who want the old behaviour, and "exit" to use the i3-nagbar to confirm exit.

That way, the default config (which most people use as template for their customizing) would confirm and if somebody really wants a silent exit, he could use "exit-without-confirm" to reflect the current behaviour.
