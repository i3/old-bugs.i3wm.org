comment:6 Changed 11 months ago by michael

Here’s a few observations:

  • You have a for_window line in your configuration that may be interfering with layout restoring: for_window [title="Spotify Premium"] floating enable; resize grow width 50 px or 50 ppt ; resize grow height 50 px or 50 ppt; move left 50 px; move up 50 px (just a guess, but worth checking out).
  • All windows that are opened in the logfile are correlated to a specific container due to their window ID, which is exactly expected behavior when restoring. From that POV, i3 definitely realizes these windows need to be put into the correct place.
  • The logfile contains a relatively large number of windows. This makes it really hard to understand/reproduce it. Could you eliminate some windows from your setup until you arrive at a minimum working example? The same goes for configuration directives, i.e. try to reproduce it with the i3 default config, then add entries until the problem reappears to narrow down what’s the problem.
