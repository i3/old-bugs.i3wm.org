#1385 new defect

While moving Eclipse tabs, tabs do not split the inner window in half as intended.

Reported by: nveeser@… Owned by:
Priority: major Component: general
Version: 4.8 Keywords:
Cc: Is it reproducible?: Always
git version: 4.8 (2014-06-15, branch "4.8") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

This is an extension on #1013
I upgraded to 4.8 and the window corruption went away, this is awesome.

But the intended behavior is still not there.

As you move the tab to the right (or left) it currently forms to transparent frames which adjust based on how far over you go. Once you let go, it should split the center window into two groups of tabs. The result is nothing happens.

Looks like there is an event that is supposed to be passed to eclipse about that which does not happen (though it's just a guess, I know little about X).

comment:1 Changed 4 months ago by abe@…

In the faq I posted a temporary workaround ​https://faq.i3wm.org/question/5060/

"detach all editors from Eclipse (it requires trial and error, as it's not clear in which direction to drag them to detach them). Once detached, they can be unfloated and it becomes super comfortable, because the windows are tiled and can be manipulated using the i3 shortcuts."

The "unfloating" is lost if you reload i3, which is quite a mess if you have a lot of editors.
