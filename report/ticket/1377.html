Description

I have lines like the following in my ~.i3/config:

bindsym $mod+1 workspace 1; exec wsinv
bindsym $mod+2 workspace 2; exec wsinv
bindsym $mod+3 workspace 3; exec wsinv

The intent is to execute the wsinv script[1] whenever I switch workspaces. This works well, except that when I login and i3 starts, workspace 1 is renamed to "1; exec wsinv". The odd name disappears (and my script gets run normally) when I switch away from, then back to workspace 1, but only if I have no applications running in it.

However, if I forget the switching away and back, and start an application in workspace 1 right after starting i3, the faulty name 1; exec wsinv 'sticks', and I can no longer switch to this renamed workspace using its keybinding anymore, I have to click on the name in the bottom bar. If I then exit any applications and switch away then back, the problem disappears.

I have tried changing the ;s in the keybindings to ,s, tried changing "exec wsinv" to "exec --no-startup-id wsinv", but to no avail.

[1]: If you're curious, wsinv is a little helper I hacked up to invert the colors on particular workspaces (configured in a separate file) so I can run browsers (and any other apps which are notoriously difficult to tame) in 'night-owl' mode ;-)
