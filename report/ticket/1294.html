#1294 new defect

i3 ignores changes in _NET_WM_STRUT_PARTIAL

Reported by: a.pronobis@… Owned by:
Priority: major Component: general
Version: 4.8 Keywords:
Cc: Is it reproducible?: Always
git version: 4.8 (2014-06-15, branch "4.8") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

I'm successfully using KDE's panel with i3. There is however one annoying problem. My panel appears on the top of the screen even if it is supposed to show on the bottom. The location of the panel seems to be detected based on the
_NET_WM_STRUT_PARTIAL and xprop actually shows correct values
_NET_WM_STRUT_PARTIAL(CARDINAL) = 0, 0, 0, 24, 0, 0, 0, 0, 0, 0, 0, 1599
(the 4th non-zero value corresponds to the bottom of the screen) as described here:
​http://standards.freedesktop.org/wm-spec/1.3/ar01s05.html

Further investigation revealed that the property is not set correctly when the window is created (STRUT_PARTIAL contains only zeros then) and soon changed to the value shown below. However, that change is not picked up by i3. Could we have handlers for changes in STRUT_PARTIAL as well just like there are for e.g. window title?

comment:3 Changed 8 months ago by lolilolicon@…

The EWMH spec says about _NET_WM_STRUT_PARTIAL:

The client MAY change this property at any time, therefore the Window Manager MUST watch for property notify events if the Window Manager uses this property to assign special semantics to the window.

I would like this too, for a different situation: it could be useful for, e.g. when you want to dock a window to a reserved area at an edge of the screen, long after it's created, by using xprop to set this property.
