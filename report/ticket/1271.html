Sorry for taking so long to get back to you.

I think there are multiple problems here:

  1. The width/height/x/y that are specified in the “geometry” property are not propagated to the “rect” property of the container when loading the layout.
  1. x/y always seems to be 0 when using i3-save-tree. Probably we need to store more than just the geometry property for floating containers.
  1. Once we fixed the above points, we’ll also need a test case that verifies that the size of floating windows does not change when saving the layout and restoring.

As a stop gap, the attached patch should work.
