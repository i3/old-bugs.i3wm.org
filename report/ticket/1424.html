Description

Hi, first of all I'm not sure if this is an i3 issue or a Firefox/Chrome issue, but I thought I'd report it here first :)

There exists a javascript API for web pages to be notified if they've been "hidden". You can test it here: ​http://ie.microsoft.com/testdrive/Performance/PageVisibility/Default.html. In other desktop environments, it becomes "hidden" when the browser is minimized or sent to an offscreen workspace. In i3, it does not. They might be looking for some kind of WM hint?

Note that browsers themselves use the visibility state not just for this JS API, but for throttling resource usage, so making this work could have benefits for battery life and memory/CPU usage.

Thanks!

comment:4 Changed 3 months ago by Ingo Bürk <admin@…>

According to ​http://standards.freedesktop.org/wm-spec/1.3/ar01s05.html:

_NET_WM_STATE_HIDDEN should be set by the Window Manager to indicate that a window would not be visible on the screen if its desktop/viewport were active and its coordinates were within the screen bounds. The canonical example is that minimized windows should be in the _NET_WM_STATE_HIDDEN state. Pagers and similar applications should use _NET_WM_STATE_HIDDEN instead of WM_STATE to decide whether to display a window in miniature representations of the windows on a desktop.

Based on that, I think it is correct not to set this flag as its viewport is not active.
