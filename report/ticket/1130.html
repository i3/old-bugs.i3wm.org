Feature: i3bar prints error message when status command fails

Description

When supporting i3 in the irc chat, I've noticed that the most common question relates to a failing status command. This is usually after the user has tried to implement a custom script or status wrapper.

There are many different ways a status command can fail. Sometimes the file cannot be opened. Sometimes the JSON is malformed. Sometimes they hack their way around the issue and we never figure out what happened.

Anyhow, customizing i3status seems to be a perilous activity. i3bar should help people identify problems by displaying a useful error message in the status line when things go wrong instead of failing silently.

This ticket was fixed in commit ​http://c.i3wm.org/bfe32ad7:

i3bar: Print error message when status_command fails

Add a function to i3bar to print an error message in the status line
when the child process invoked by status_command fails to provide
input that can be displayed as a statusline.

When the child provides JSON that cannot be parsed, alert the user and
convey a short message provided by yajl communicating the specific
problem.

When the child (or the shell executing the status command) exits
unexpectedly, alert the user and display the exit code. The cases where
the status command is not executable or not found in the user's PATH are
treated specially.

fixes #1130
