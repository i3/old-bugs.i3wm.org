Reported by: jasper@… Owned by: michael

Description

After updating from i3 4.3 to 4.4 i3status has been crashing and moving the statusbar icons (ie, pidgin/gmpc) into their own windows on my desktop.

This doesn't happen when using a single screen, only with multiple screens.

This is my i3 config which triggers the issue:

# i3 config file (v4)

font -misc-fixed-*-*-*-*-10-100-75-75-c-60-iso10646-1

# start a terminal
bindsym F2 exec /home/jasper/bin/term.sh

# switch to workspace
bindsym Mod1+1 workspace 1
bindsym Mod1+2 workspace 2
bindsym Mod1+3 workspace 3
bindsym Mod1+4 workspace 4
bindsym Mod1+5 workspace 5
bindsym Mod1+6 workspace 6
bindsym Mod1+7 workspace 7
bindsym Mod1+8 workspace 8
bindsym Mod1+9 workspace 9

# desktop (bar + i3status + tray on left monitor, and regular bar on the right)
bar {
	# prepend i3status output with current MPD song.
	status_command /home/jasper/bin/i3status_mpd.sh
	output DVI-1
}

The issue can be triggered by:
Have one icon in the i3status area, for example pidgin. The switch to the workspace that has pidgin, now switch to another and switch back...boom. It's not necessary to reproduce that the workspaces are on different monitors. The issue occurs also when I switch between workspaces on the same monitor, as long as I run i3 on both monitors.

When I remove "output DVI-1" from my config the issue doesn't occur.

comment:21 Changed 2 years ago by Michael Stapelberg

  • Resolution set to fixed

This ticket was fixed in commit ​http://c.i3wm.org/f5b7bfb1:

Bugfix: fix IPC messages writes with low buffer sizes (Thanks jasper, dcoppa)

Use the following command to reproduce this bug:

    echo 4096 | sudo tee /proc/sys/net/core/wmem_default

Then just switch workspaces with some windows on it and i3bar would
exit due to malformed IPC messages.

This bug hits OpenBSD users (and possibly other BSDs) due to their lower
default buffer size.

fixes #896
