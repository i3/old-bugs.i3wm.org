Description

I'm running i3 together with KDE 5.0. The problem I'm observing is that every second time I open a kicker window (the one shown in the image here: ​http://regmedia.co.uk/2014/07/28/kde_plasma_5_kickoff_menu_resized.jpg), it appears in the middle of the screen instead of it's normal location.

I'm opening and closing the window using the (K) button in the left bottom corner. Basically, every 2 clicks on the (K) button, the window appears in the center of the screen and then next time in it's correct position and so on.

However, I have noticed that if the window is closed by my i3 shortcut, the problem does not occur and the window is re-created in it's correct location every time.

Moreover, even if the window is not shown, restarting i3, will leave an "empty" window hanging in the middle of the screen. It seems that KDE does not really close the window, but hides it somehow.

This problem does not occur for current master branch, but only for next. I bisected it to the following commit:
43b447855df095c56e860117d53acd0dd09ac42b
When this commit is reverted, the problem does not occur even in the next branch.
