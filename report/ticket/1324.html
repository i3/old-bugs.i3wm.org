Description

i3 currently explicitly features the ability to cycle forwards/backwards through current floating windows with the focus left/right command.

On my current workflow password dialogs can popup sometimes heavily delayed, which can cause my freshly popped up password dialog of some other process to be overlapped by a previous password dialog that I decided tho to finish after I entered the password for my freshly spawned process.

As there already seems to be intent to change back to previous floating windows, I propose that these commands also promote the newly focused window to the top, as my password dialogs always center and thus overlap.

Further there's a specific code-comment at focusing floating up/down in the current code-base with:

/* TODO: implement cycling through the z-index */

Is this still planned?

If this is not i3s purpose, can someone explain what purpose the current implementation of changing the floating focus has?

Thanks for any response.
