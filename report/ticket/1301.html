unmapping and remapping a floating window loses position information

Reported by: i3@… Owned by: TonyC
Priority: major Component: general
Version: 4.8 Keywords:
Cc: Is it reproducible?: Always

Description

In version 4.7.2, I could create a floating window, move it to the corner of the screen, and then have it unmap and remap itself, and it would stay in the corner of the screen. In version 4.8, it moves back to the center of the screen every time it is remapped. Bisecting tracked this down to commit 00ee86de, but it's not really clear how that commit is related.

comment:2 Changed 9 months ago by i3@…

Unmapping and remapping just means hiding and showing the window. The specific application I'm using is rxvt-unicode, with the urxvt-kuake extension enabled. I run urxvt -pe "kuake<F2>" -name urxvt-kuake from my .xinitrc, and then have

for_window [instance="urxvt-kuake"]     floating enable
for_window [instance="urxvt-kuake"]     move absolute position 0 px 0 px

in my .i3/config file. Previously, this would cause the terminal to show up at the top left of the screen every time I pressed F2, but now it always shows up in the center of the screen. Even if I run i3-msg "move absolute position 0 px 0 px", it will move to the correct location, but if I press F2 to hide it and then F2 to show it again, it will be back in the center.

comment:5 Changed 9 months ago by michael

Oh, sorry, I missed that. My guess is that the changes to src/manage.c are what’s causing the problem (i.e. the different position in which focus will be set). This in turn is in combination with a bug in cmd_move_window_to_position(), which always operates on the focused window instead of the windows that match the specified criteria. See c2b6b06da7db49bd3ecd6fa1561ef2b407ff64a3 for an example of how to fix that. Patches greatly appreciated (see ​http://cr.i3wm.org/).
