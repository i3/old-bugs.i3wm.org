Combined geometry hint and resize does not work as expected

Reported by: phillip.berndt@… Owned by: michael

Version: 4.5.1 Keywords:

git version: 4.5.1 (2013-03-18, branch "tags/4.5.1") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

If a floating window changes the aspect in its geometry hint, i3 gradually scales down the window to fit the new requirements. If during this shrinking the window requests a resize, i3 performs it but then continues to shrink the window to the size it originally tried to reach (which usually is quite tiny).

Other WMs get this right. (Tested: xfwm4, mutter, muffin)

Example code

#include <gtk/gtk.h>
#include <gtk/gtk.h>
#include <gtk/gtk.h>

GtkWidget *wnd;

gboolean change_aspect(gpointer foo) {
        GdkGeometry hints;
        hints.min_aspect = hints.max_aspect = 640./480.;
        gtk_window_set_geometry_hints(GTK_WINDOW(wnd), NULL, &hints, GDK_HINT_ASPECT);
        gtk_window_resize(GTK_WINDOW(wnd), 640, 480);

        return FALSE;
}

int main(int argc, char *argv[]) {
        gtk_init(&argc, &argv);

        wnd = gtk_dialog_new();

        GdkGeometry hints;
        hints.min_aspect = hints.max_aspect = .5;
        gtk_window_set_geometry_hints(GTK_WINDOW(wnd), NULL, &hints, GDK_HINT_ASPECT);

        gtk_widget_show(GTK_WIDGET(wnd));

        g_timeout_add(1000, change_aspect, NULL);

        gtk_main();
}

gcc pkg-config gtk+-2.0 --cflags --libs hints.c && ./a.out

Expected

Window pops up and changes its size to 640x480 after one second

Actual result

Window changes its size to 640x480 after one second but then gradually shrinks to 30x22
