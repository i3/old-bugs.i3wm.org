"bspwm-like" simultaneous x and y axis manipulation with mouse

Description

In bspwm it's possible, when using $mod+ Right Click, to manipulate both the x axis and the y axis at once when resizing containers, whereas i3 only allows the manipulation of one axis. While i3 obviously is a keyboard-driven WM, sometimes using the mouse is just convenient, as I believe would be the case with this.

Example:
​http://youtu.be/U17CLayt_aA?t=14s (14 seconds in)

Would this be possible to implement?

comment:3 Changed 4 months ago by Ingo Bürk <ingo.buerk@…>

This might actually not be that easy. It certainly isn't as easy as with floating windows (which might just be the reason it's different for them).

Consider a layout hsplit(term1, vsplit(term2, term3)).

First of all, you need to make sure whether resizing in both directions makes sense. Resizing the inner corners of term2 or term3 does, but resizing in both x/y direction for term1 never makes sense.

Also, when resizing the top left corner of term 3, you need to both update the relation between term2 and term 3 and the relation between term1 and the vsplit container. So you resize on multiple levels in the tree.

This might be tricky.
