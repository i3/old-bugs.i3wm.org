comment:6 Changed 3 years ago by anonymous

It seems like the cause for this problem is that chrome/chromium doesn’t think it’s active. It checks/watches the _NET_ACTIVE_WINDOW property on the X11 root window, which i3 seems to set correctly. The appropriate place in the Chrome source code is src/ui/base/x/active_window_watcher_x.cc

comment:7 Changed 3 years ago by anonymous

Further investigation revealed that i3 sets the _NET_SUPPORTING_WM_CHECK to the root window. This will cause chrome to change its eventmask to StructureNotify (only!) on the root window, effectively overriding the previous eventmask of StructureNotify,PropertyChange. Therefore, chrome will not get PropertyNotify events for the _NET_ACTIVE_WINDOW changes. Manually changing the _NET_SUPPORTING_WM_CHECK to a different (existing) window fixes the problem.
