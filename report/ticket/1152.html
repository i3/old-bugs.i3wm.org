Cross-output tile movement sometimes causes the moving tile to lose it's select/active status

Reported by: i3-new-ticket Owned by: TonyC

Description

Scenario: I have 3 outputs for 3 monitors. Output 1, 2 and 3. All of them are full of tiles. There is an output (3rd one) that is cursed (dunno why).

Creating the bug:

1- Select a tile in output 1.
2- Keep moving it to the right (i.e. towards output 2) using shift+mod+rightarrow.
3- The tile will eventually enter output 2, and remain selected (totally great).
4- Keep shift+mod+rightarrow until it enters the cursed output (i.e. output 3).
5- Wheneer the tile enters the cursed output (i.e. output 3), the tile loses its select/active status. I.e. the tile enters output 3, but the selected/active tile would be switched into some other tile in output 2.
