IPC: "empty" event should have the info of which workspace is emptied

Reported by: hong@… Owned by: TonyC
Priority: major Component: general
Version: 4.8 Keywords: easy
Cc: Is it reproducible?: Always
git version: 4.8 (2014-06-15, branch "4.8") Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

Currently there is no info of which workspace is emptied in the "empty" event.

Tracking the old focused workspace isn't reliable, as windows in other workspaces can also be killed to empty another workspace without switching to it at all. It would be a really useful information of which workspace is emptied.

comment:2 Changed 4 months ago by TonyC

We don't have any great options in my opinion for implementing this without making breaking changes to the api, but we can make it work.

I would like to keep the object signatures the same for every workspace event, so I don't want to add any new properties. Unfortunately, whoever implemented the properties "current" and "old" did not make the names general enough to perfectly fit with the other kinds of events.

I think the best way to go here is to use the "current" property just as though we would use the "container" property on the window events. On each of the workspace events, this will be the affected container for the event (the focused, emptied, urgent, or initted container respectively for each workspace event).

What do you think?
