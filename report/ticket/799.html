Description

The scenario: two monitors, A with workspace 1 and B with workspace 2; pointer is in A/1. Upon sending the i3 command "workspace 2", focus is changed to B/2 and the mouse is moved to the middle of workspace 2.

This is annoying because I rarely use the mouse, but if I'm frequently switching focus among monitors, I have to move the pointer out of the way every time.

I think there are three possible remedies:

  • disable this automagic pointer movement altogether;
  • a configuration knob to disable it;
  • or for i3 to remember where the mouse was last placed on each monitor and to return it there when focus is shifted to it.

Also, I don't see how the existing behavior scales with multiple pointers. In my mind, a desire to support multiple pointers would be an argument for the third suggested option.

Fixed by â€‹http://code.stapelberg.de/git/i3/commit/?h=next&id=b109b1b20dd51401dc929407453d3acdd8ff5566

Disable render-time pointer warps if asked

When `focus_follows_mouse` configuration option is disabled, do not warp
the pointer when focus changes outputs after rendering.

Rationale: this option is meant to be disabled by users who have a setup
where the mouse is usually in the way. These users tend to move the
mouse to a corner or use a utility to hide the pointer when it is not
active. When this user switches focus between outputs, the mouse is
placed in the center of the screen.

This is clearly against the spirit of disabling `focus_follows_mouse`.
Disabling this option now implies disabling "mouse follows focus".
