Dock windows should be stacked at the top edge of the screen

Reported by: kaze@… Owned by: michael
Priority: major Component: feature-request
Version: Keywords: dock systray desktop-panels
Cc: Is it reproducible?: Always
git version: Operating system: Affects all
X11 program: Distribution / flavor:
Your IRC nick:

Description

All dock windows are stacked on the bottom edge in every workspace, just above the workspace bar. This is painful when dealing with ​dzen2 -dock, ​trayer or desktop panels like ​fbpanel, ​xfce4-panel, ​gnome-panel, ​kicker, etc.

Screen edges are very easy to reach with the mouse, that’s why they’re so valuable. Since the i3 workspace bar is at the bottom of the screen, I think i3 should stack dock windows at the top edge by default.

Feature request: implement a mechanism to specify on which screen edge dock windows should be stacked. There’s a similar stuff in Xmonad (see the ​ManageDocks plugin), and it’s pretty neat!

Having the ability to display a desktop panel in i3 would be a nice workaround to ​ticket #28.
